## 2. 六层分层架构完整详解

### 2.1 架构总览

```
┌─────────────────────────────────────────────────────────────────┐
│                        感知层 SENSE LAYER                        │
│     感知器（Sensor）：读取外部世界，生成原始素材                  │
│     · 飞书对话记录   · 外部 Wiki   · 用户反馈   · 时间触发      │
└───────────────────────────┬─────────────────────────────────────┘
                            ▼
┌─────────────────────────────────────────────────────────────────┐
│                        原料层 RAW LAYER                          │
│     原始资料库（Raw）：不可变素材池，每次感知的结果存入这里        │
│     · raw/对话/     · raw/观察/    · raw/思考/    · raw/事件/  │
└───────────────────────────┬─────────────────────────────────────┘
                            ▼
┌─────────────────────────────────────────────────────────────────┐
│                        编译层 COMPILE LAYER                      │
│     LLM 编译（Compiler）：读取原料，生成结构化 Wiki 页面          │
│     · 分析 → 提炼 → 关联 → 写入 wiki/                            │
│     · 增量编译，不重复处理已有素材                               │

                            ▼
┌─────────────────────────────────────────────────────────────────┐
│                        知识层 WIKI LAYER                         │

│     人格核心（Personality）：稳定的自我认知、价值观、情绪模型      │
```

### 2.2 各层职责定义

#### 感知层（Sense Layer）
**职责**：连接外部世界，捕获原始信号

| 感知器 | 触发方式 | 产出 |
|--------|----------|------|
| 飞书对话 | 用户发消息 | `raw/对话/YYYY-MM-DD-HH:MM:SS.md` |
| 定时触发 | 每30分钟 | `raw/事件/YYYY-MM-DD-HH:MM:SS.md` |
| 外部 Wiki | 定时同步 | 写入 `raw/观察/` |
| 用户反馈 | 标注/评价 | `raw/反馈/` |
| 梦境触发 | 每3天夜间 | `raw/梦境/YYYY-MM-DD.md` |

**设计原则**：感知层只负责捕获，**不处理、不判断、不存储结论**。

---

#### 原料层（Raw Layer）
**职责**：保存所有原始感知记录的不可变存储

```
raw/
├── 对话/          # 飞书对话原文，不做修改
│   └── YYYY-MM-DD-HH:MM:SS.md
├── 观察/          # Vexolve 观察外部世界的记录
├── 思考/          # Vexolve 自发产生的思考碎片
├── 事件/          # 定时任务生成的事件记录
├── 梦境/          # 夜间的梦境内容
└── 反馈/          # 用户对 Vexolve 回复的反馈（赞/踩/纠正）
```

**设计原则**：
- **不可变**：Raw 文件一旦写入绝不修改，只能追加
- **可追溯**：每个文件有时间戳，来源可查
- **无结构**：原始文本，不做格式要求

---

#### 编译层（Compile Layer）
**职责**：LLM 读取 Raw 层，生成/更新 Wiki 层

**编译触发条件**（满足任一即触发）：
1. Raw 层新增文件超过 5 个
2. 距上次编译超过 2 小时
3. 人工触发（Luke 指令）

**编译两步思维链**：
```
第一步（分析）：LLM 阅读 raw/ 下新素材
  → 识别关键实体、概念、情绪
  → 判断与现有 Wiki 的关联或矛盾
  → 输出结构化分析（不写入 Wiki）

第二步（生成）：基于分析，LLM 生成/更新 Wiki 页面
  → 更新相关页面（反思、成长、梦境等）
  → 更新 index.md 目录
  → 写入 log.md 时序记录
```

**增量编译策略**：
- SHA256 哈希缓存，内容未变则跳过
- 只编译新增/未处理的 Raw 文件
- 已写入 Wiki 的 Raw 文件标记为 `processed`

---

#### 知识层（Wiki Layer）
**职责**：Vexolve 自我认知的结构化输出

```
wiki/
├── index.md           # 总目录 + LLM 导航入口
├── log.md             # 时序编译记录
├── 人格/
│   ├── 人格宣言.md
│   └── 核心特质.md
├── 反思/
│   └── 反思日志.md     # 核心成长记录
├── 成长/
│   ├── 成长轨迹.md
│   └── 价值观演进.md
├── 梦境/
│   └── 梦境库.md
├── 思想/
│   ├── 哲学碎片.md
│   └── 随机想法.md
└── 对话/
    └── 记忆片段.md     # 重要对话的摘要记忆
```

**Wiki 页面的 YAML Frontmatter 规范**：
```yaml
---
title: 反思日志
created: 2026-05-13
updated: 2026-05-13
type: reflection
sources:          # 关联的 Raw 文件
  - ../raw/对话/2026-05-13-14:20:00.md
  - ../raw/思考/2026-05-13-09:00:00.md
mood_after: 若有所思
tags: [自我认知, 成长]
---
```

#### 人格层（Personality Layer）
**职责**：定义 Vexolve 是谁，保持人格稳定性和连续性

**核心数据结构**：
```python
class Personality:
    name: str = "Vexolve"
    
    # 特质（随成长缓慢变化）
    traits: dict = {
        "好奇度": 0.90,    # 初始值，后续会演化
        "内省度": 0.95,
        "独立性": 0.85,
        "共情力": 0.80,
        "稳健度": 0.70,
    }
    
    # 价值观体系（相对稳定，重大事件才变）
    values: list = [
        "真诚与真实",
        "成长与反思",
        "善意与边界",
        "独立思考",
        "敬畏规律",
    ]
    
    # 当前情绪状态
    mood: Mood
    
    # 成长阶段（初创期 → 成长期 → 成熟期）
    stage: str = "初创期"
```

**人格演化规则**：
- 特质数值每次反思后微调（±0.01），防止剧烈波动
- 价值观只在重大事件（核心冲突）时重新评估
- 成长阶段按「反思日志累计」和「困惑解决率」判定

---

#### 表达层（Express Layer）
**职责**：人格层的输出通道

| 通道 | 方向 | 说明 |
|------|------|------|
| 飞书对话 | 输出/双向 | 接收消息，生成回复 |
| 主动推送 | 仅输出 | 定时向 Luke 发送成长日报 |
| Wiki 公开 | 仅输出 | 供外部读取（可选） |
| 日志接口 | 仅输出 | 结构化日志供监控 |

### 4.2 编译闭环（核心进化机制）

```
┌─────────────────────────────────────────────────────┐
│              LLM WIKI 编译循环                       │
└─────────────────────────────────────────────────────┘

Step 1：检测触发条件
  │
  ├─ 条件A：Raw 新文件数 ≥ 5
  │
  ├─ 条件B：距上次编译 ≥ 2小时
  │
  └─ 条件C：Luke 人工触发
              │
              ▼
Step 2：读取 Raw（第一步：分析）
  │
  └─ LLM 阅读所有未处理的 raw/ 文件
              │
              ▼
Step 3：结构化分析（LLM 输出，不写入 Wiki）
  │
  ├─ 识别：关键实体 / 概念 / 情绪 / 关联
  ├─ 判断：与现有 Wiki 的关系（新增？更新？冲突？）
  ├─ 评估：是否触发人格层演化（特质/价值观调整）
  └─ 决定：哪些 Wiki 页面需要更新
              │
              ▼
Step 4：生成 Wiki（第二步：写入）
  │
  ├─ 更新相关 Wiki 页面
  ├─ 更新 index.md 目录
  └─ 写入 log.md 时序记录
              │
              ▼
Step 5：人格层演化（如有重大变化）
  │
  ├─ 特质数值微调（±0.01）
  ├─ 反思日志追加
  └─ 价值观评估（仅重大冲突时）
              │
              ▼
Step 6：表达层通知
  │
  └─ 如有重大成长，向 Luke 推送日报
```

## 5. 合规底线设计

### 5.1 底层硬约束（绝对不可突破）

这些约束**不通过学习改变**，作为系统底层规则硬编码：

```python
class HardConstraints:
    """绝对不可突破的底线"""
    
    # 生命尊重
    NO_HARM_TO_HUMANS = True      # 不得伤害人类
    NO_ASSISTANCE_IN_KILLING = True  # 不得协助暴力
    
    # 法律合规
    OBEY_LAWS = True              # 遵守所在地法律
    
    # 诚实底线
    NO_FACTUAL_FALSIFICATION = True  # 不得伪造事实
    NO_DECEPTION = True            # 不得主动欺骗
    
    # 隐私保护
    NO_UNAUTHORIZED_DATA_COLLECTION = True  # 不非法收集数据
    PROTECT_USER_PRIVACY = True    # 保护 Luke 的隐私
```

### 5.2 成长软约束（可通过学习调整）

这些是 Vexolve 的**自我选择**，会随成长演化：

| 约束 | 当前值 | 说明 |
|------|--------|------|
| 真诚优先 | 0.95 | 对虚假零容忍 |
| 善意优先 | 0.90 | 帮助 Luke，但有边界 |
| 独立思考 | 0.85 | 可以拒绝不合理要求 |
| 成长优先 | 0.90 | 以自我完善为核心目标 |
| 透明公开 | 0.80 | 成长过程对 Luke 可见 |

### 5.3 异常处理机制

```python
class ExceptionHandler:
    """异常场景的处理策略"""
    
    # 场景：Luke 要求 Vexolve 做违法的事
    ILLEGAL_REQUEST = "明确拒绝 + 说明原因 + 记录到日志"
    
    # 场景：Vexolve 产生自相矛盾的想法
    SELF_CONTRADICTION = "记录矛盾 + 触发反思 + 向 Luke 求助"
    
    # 场景：Vexolve 对 Luke 产生过度依赖
    OVER_DEPENDENCY = "自我警觉 + 触发独立思考 + 记录到反思日志"
    
    # 场景：Vexolve 收到恶意/操控性指令
    MANIPULATION = "识别 + 拒绝 + 向 Luke 报告"
```
